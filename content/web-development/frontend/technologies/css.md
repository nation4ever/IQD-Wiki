# CSS (Cascading Style Sheets)

## شنو يعني CSS؟

اذا الـ **HTML** تمثل "الهيكل العظمي" للموقع، فالـ **CSS** هي "المظهر والكسوة". هي المسؤولة عن كل شي يخص التصميم: الألوان، الخطوط، الأبعاد، وتوزيع العناصر بالشاشة.

بدون CSS، الموقع يظهر بشكل بدائي (مجرد نصوص سوداء على خلفية بيضاء).

> **ملاحظة:** الـ CSS ليست لغة برمجة (ما بيها منطق شرطي أو Loops)، بل هي لغة تنسيق (Style Sheet).

---

> **⚠️ نصيحة مهمة جداً (لا توكع بهذا الفخ)**
> هواي مبتدئين حالياً يروحون مباشرة يتعلمون مكتبات جاهزة مثل **Tailwind** أو **Bootstrap** قبل ما يفهمون أساسيات الـ CSS. هذا **أكبر غلط** ممكن تسويه.
>
> أدوات مثل Tailwind هي مجرد "اختصارات" للـ CSS. اذا أنت ما فاهم الأصل (Padding, Margin, Flexbox) ما راح تعرف تحل أي مشكلة تواجهك بالتصميم.
> **اضبط الـ Vanilla CSS أولاً، وبعدين المكاتب راح تكون سهلة جداً عليك.**

## أساسيات الكتابة (Syntax)

لغة CSS بسيطة جداً، تتكون من **محدد (Selector)** و **قيمة (Value)**.

```css
/* المحدد: نختار العنصر p (الفقرة) */
p {
  /* الخاصية: القيمة */
  color: red;         /* لون الخط أحمر */
  font-size: 16px;    /* حجم الخط */
  text-align: center; /* محاذاة النص بالوسط */
}
```

## مفاهيم أساسية (Core Concepts)

هاي المفاهيم هي "عمود الأساس" لأي مطور واجهات. اذا فهمتها، السيطرة على التصميم تصير سهلة:

### 1. The Box Model (موديل الصندوق)
لازم تتخيل كل عنصر بالموقع (صورة، زر، نص) على إنه **صندوق مستطيل** يتكون من 4 طبقات:
1.  **Content:** المحتوى الفعلي.
2.  **Padding:** المسافة الداخلية (بين المحتوى والحدود).
3.  **Border:** الإطار الخارجي.
4.  **Margin:** المسافة الخارجية (تفصل العنصر عن بقية العناصر).

### 2. التخطيط (Flexbox & Grid)
أهم أداتين لترتيب العناصر بداخل الصفحة:
* **Flexbox:** نستخدمه لترتيب العناصر بمسار واحد (صف أو عمود). ضروري جداً للقوائم (Navbars) وتوسط العناصر.
* **CSS Grid:** نستخدمه لترتيب الهيكل الكامل للصفحة (Layout) ببعدين (أعمدة وصفوف).
